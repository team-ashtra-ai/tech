Legal

Robots and crawl guidance

This page explains how the public robots rules support a static-first, crawlable site without hiding the published content.

Go to sitemap Legal notice
01

Contexto

Introduction

rotata.es publishes a simple public crawl policy intended to help search engines discover the site efficiently.

02

Contexto

Purpose of robots.txt

robots.txt gives search engines a public set of crawling instructions and points them to the sitemap used for discovery.

03

Modelo operativo

Allowed paths

The public build is intended to be crawlable by default.

Node 01

Published HTML routes

Node 02

Public styles, scripts and image assets required for rendering

Node 03

Blog article routes and legal pages

Node 04

The XML sitemap published at the domain root

04

Modelo operativo

Disallowed paths

The current public robots policy does not publish route blocks by default.

Node 01

No public route blocks in the standard build

Node 02

Unpublished source files are not exposed

Node 03

404 pages are excluded from the sitemap

Node 04

Future exclusions should be declared in source and deployed through the build

05

Contexto

Crawl delay

rotata.es does not publish a crawl-delay directive in its default robots output.

06

Modelo operativo

Sitemap reference

Search engines receive a direct reference to the XML sitemap.

Node 01

robots.txt points to https://rotata.es/sitemap.xml

Node 02

The sitemap is regenerated during the build

Node 03

The sitemap is published at the domain root

Node 04

The sitemap lists public HTML routes intended for discovery

07

Contexto

Search engine notes

Search engines are expected to crawl the published HTML, respect redirects and index the public site structure unless future source rules state otherwise.

08

Modelo operativo

Indexing policy

The site is published as crawlable static output.

Node 01

Core pages are intended for indexing

Node 02

Legal pages remain publicly accessible

Node 03

Assets needed to render the pages are available

Node 04

Redirects are shipped through build artefacts when routes change

09

Contexto

Updates / changes

Any change to crawl rules, sitemap output or indexing policy should be made in source control and deployed through the build so Cloudflare and WordPress exports remain aligned.

10

Modelo operativo

Contact

Questions about crawling or indexing can be sent through the published contact routes.

Node 01

Email: info@rotata.com

Node 02

Contact page: /contact/

Node 03

Sitemap page: /legal/sitemap/

Node 04

Legal notice: /legal/

Use contact for an indexing question
