Contexte

Commitment statement

Rotata works to make rotata.es usable by keeping structure, contrast and interaction clear.

Modèle opératoire

Accessibility standards

The site aims to follow WCAG 2.1 AA-aligned practices where practical for its static-first architecture.

01

Semantic HTML

02

Keyboard-accessible navigation

03

Visible focus states and readable contrast

04

Responsive layouts supporting zoom and reflow

03

Modèle opératoire

Scope of compliance

Accessibility work currently focuses on the core published experience.

01

Main page templates and navigation

02

Contact and consultation forms

03

Cookie consent and utility widgets

04

Legal and informational pages

04

Couches du système

Measures implemented

Accessibility depends on repeatable implementation choices.

Structured headings

Pages follow a clear heading hierarchy and semantic landmarks.

Labeled forms

Form controls use visible labels and validation messaging.

Keyboard access

Navigation and main interactions are reachable without a mouse.

Readable mobile layout

Text, spacing and tap targets are designed for small screens first.

05

Modèle opératoire

Known limitations

Some areas still need ongoing review and refinement.

01

Third-party scripts may introduce behaviour outside Rotata’s direct control

02

Decorative visuals are not relied on for essential meaning

03

Content updates require periodic accessibility QA

04

Future integrations should be reviewed before being treated as fully accessible

06

Modèle opératoire

Assistive technologies supported

The site is designed to work with common browser and device accessibility features.

01

Keyboard-only navigation

02

Browser zoom and responsive reflow

03

Modern browser-based screen reader usage

04

Reduced-motion and standard mobile accessibility settings

07

Contexte

Feedback mechanism

Users can report accessibility issues, unclear interactions or readability problems so Rotata can review them directly.

08

Cadence

Improvement roadmap

Accessibility issues should move through a visible correction loop.

  1. 01Receive and reproduce the issue
  2. 02Assess impact and root cause
  3. 03Fix the source component or content pattern
  4. 04Retest before release
09

Modèle opératoire

Review / audit process

Accessibility quality improves when review is repeated at component and page level.

01

Manual keyboard passes

02

Contrast and readability checks

03

Form review for labels and feedback states

04

Regression review after site updates
