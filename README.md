# Rotata Static Website

Premium static-first website for `rotata.es`, positioned as a B2B growth system company: **the system is the product; tools support the system**.

## Local Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
npm run build
npm run validate
```

The public build is generated into `dist/` for Cloudflare Pages.

## Structure

- `src/content/` stores editable multilingual page and blog content.
- `src/data/` stores the published information architecture in `navigation.json`, source page copy in `site.json`, plus partners, forms, cookie text and scraped source metadata.
- `src/data/showcase-sites.json` stores the five standalone concept site registry used by the header preview bar.
- `src/data/themes.json` stores legacy token theme metadata for future URL-generated experiments.
- `src/partials/` stores reusable header, footer, forms, CTAs, cards, cookie, SEO and schema partials.
- `src/partials/theme-sites/` stores standalone concept site templates for the website showcase.
- `src/assets/logo/` stores reusable light/dark brand marks plus exported raster logo files for static and WordPress builds.
- `src/assets/profile/` stores reusable profile cutouts and generated portrait variants when they are added to the site.
- `src/styles/` stores the premium design system.
- `src/styles/theme-sites/` stores independent CSS for the standalone concept sites.
- `src/styles/themes.css` stores the base theme presets, while `src/styles/generated/theme-overrides.css` stores URL-generated theme slot overrides.
- `src/assets/section-visuals/` stores generated hero and section artwork built from Rotata's image library.
- `src/assets/theme-sites/` stores generated concept assets built by `scripts/generate_showcase_assets.py`, including per-concept home heroes plus exact page and section visuals for showcase detail routes.
- `src/scripts/` stores consent-aware navigation, forms, analytics, language and animation JS.
- `src/scripts/theme-sites/` stores concept-specific JS for the standalone showcase pages.
- `scripts/` stores build, SEO, schema, validation, section-visual generation, image, theme-builder, content-machine and WordPress export tooling.
- `scripts/publish_github.py` stages, commits and pushes the current branch with the current date and time as the commit message.
- `scripts/open_preview.py` starts the local dev server when needed and opens the English concept preview at `http://127.0.0.1:8000/en/showcase/databricks/`.
- `static-sites/` stores six independent no-build Cloudflare Pages roots. Each variation is plain HTML, CSS, vanilla JS, JSON and local assets, rebuilt from the authored concept exports and copied out as standalone deployable roots.
- `wordpress-export/rotata-theme/` is generated by `scripts/export_wordpress_theme.py`.

## Commands

```bash
npm run dev                # Build, watch, and serve locally at http://127.0.0.1:8000
npm run serve              # Serve existing dist/ without rebuilding
npm run build              # Build static site, sitemap and schema
npm run validate           # Validate internal links and SEO basics
npm run blog:translate     # Translate Spanish blog article JSON into en/fr article files
npm run lang:generate      # Copy English into es/fr as placeholders
npm run lang:generate:ollama # Translate en -> es/fr with local Ollama if configured
npm run lang:source:en     # Set English as the source-of-truth language
npm run lang:source:es     # Optional: switch source-of-truth to Spanish later
npm run lang:source:fr     # Optional: switch source-of-truth to French later
npm run optimize:images    # Optional Pillow-based WebP optimization
npm run og                 # Generate SVG OG images
npm run wp:export          # Generate WordPress theme folder
npm run wp:zip             # Zip WordPress theme
npm run content:ideas      # Generate article ideas/content drafts
python3 scripts/theme_builder.py https://example.com signal-paper --label "Example Theme" # Replace a theme slot from a reference URL
python3 scripts/publish_github.py # Add, commit, and push with a timestamp message
python3 scripts/open_preview.py   # Start the local dev server if needed and open the English concept preview
```

## Cloudflare Pages

Build command: `npm run build`

Output folder: `dist`

Environment variables can be added in Cloudflare Pages and are read at build time.

For the independent no-build concept sites in `static-sites/`, create a separate Cloudflare Pages project, set the root directory to the desired variation folder, leave the build command blank, and set the output directory to `/`. See `docs/static-site-variations.md`.

Copy `.env.example` to `.env` locally or define these in Cloudflare:

- `PUBLIC_GA4_ID`
- `PUBLIC_GTM_ID`
- `PUBLIC_LINKEDIN_PARTNER_ID`
- `PUBLIC_CLARITY_ID`
- `PUBLIC_HUBSPOT_PORTAL_ID`

- `PUBLIC_FORMS_ENDPOINT`

## Content Editing

Edit published routes, dropdowns, footer columns and page composition in `src/data/navigation.json`. Edit reusable and route-specific source copy in `src/data/site.json`. Concept demo labels, routes, reference URLs and assets live in `src/data/showcase-sites.json`. English is currently the source of truth for `site.json`, controlled by `"source_language": "en"` at the top of that file. Blog articles live in `src/content/es/blog/*.json`, with generated localized versions in `src/content/en/blog/*.json` and `src/content/fr/blog/*.json`, and are indexed newest first. Partners, cases and forms are in `src/data/`.

## Recommended Editing Workflow

1. Edit the published IA first in `src/data/navigation.json` if routes, menus, footer columns or page composition need to change.
2. Edit the English source copy in `src/data/site.json` under each source page's `translations.en` block. Published page keys can mirror `src/data/navigation.json` when a route needs its own dedicated content source.
3. Edit design and behaviour in:
   - `src/styles/*.css`
   - `src/styles/generated/theme-overrides.css` if you are reviewing URL-generated theme output
   - `src/styles/theme-sites/*.css` if you are editing standalone concept sites
   - `src/scripts/*.js`
   - `src/scripts/theme-sites/*.js`
   - `src/partials/*.html`
   - `src/partials/theme-sites/*.html`
   - `src/data/showcase-sites.json` if you need to relabel, reorder or replace concept sites
   - `scripts/build_site.py` if layout/rendering logic needs to change
   - `scripts/theme_builder.py` if you want to change how reference URLs are analyzed into theme overrides
   - `scripts/generate_section_visuals.py` if you need to retune the generated image direction
   - `scripts/generate_showcase_assets.py` if you need to retune concept asset generation
4. Run `npm run dev` and preview the showcase at `http://127.0.0.1:8000/en/showcase/databricks/`.
5. Refresh the browser after each save. The watcher rebuilds automatically.
6. When English source copy in `site.json` is final, run `npm run lang:generate` or `npm run lang:generate:ollama` if needed. CTA targets can be controlled in `site.json` with `primary_cta_page`, `secondary_cta_page`, `final_cta_page` or direct `*_cta_url` values.
7. When Spanish blog source articles change, run `npm run blog:translate` to refresh `src/content/en/blog/` and `src/content/fr/blog/`.
8. Rebuild with `npm run build`.
9. Export WordPress with `npm run wp:export && npm run wp:zip`.

Important:

- Do not edit files inside `dist/`. They are generated output.
- Do not edit files inside `wordpress-export/rotata-theme/` directly unless you are intentionally changing the export layer. Regenerate them after source changes.
- The published IA lives in `src/data/navigation.json`.
- The reusable source-of-truth content library lives in `src/data/site.json`.
- The standalone concept demo is driven by `src/data/showcase-sites.json`, `src/partials/theme-sites/`, `src/styles/theme-sites/`, `src/scripts/theme-sites/`, and `src/assets/theme-sites/`.
- Preview the English concept showcase at `/en/showcase/databricks/`; the normal English site remains at `/en/`.
- Cookie consent, WhatsApp, back-to-top and Soy Rotbot live in generated HTML from `scripts/build_site.py` and behavior in `src/scripts/consent.js` plus `src/scripts/widgets.js`. The cookie banner is a compact three-action module: accept all, read policy, or necessary only.
- Section and hero visuals are generated into `src/assets/section-visuals/` before each build, and concept assets are generated into `src/assets/theme-sites/`, so the output stays static-first and exportable.

## SEO And Schema

SEO metadata is generated from page content plus `src/data/seo.json`. Schema modules are written to `schema/` and embedded per page at build time.

## WordPress Export

```bash
npm run wp:export
npm run wp:zip
```

The export preserves assets, CSS, JS, template parts and theme support for menus, logo, thumbnails, translations and editor styles. Future client editing should move page copy into WordPress pages/posts/custom post types rather than hard-coding it in PHP.

## Troubleshooting

- If links fail, run `python3 scripts/validate_links.py`.
- If SEO fails, run `python3 scripts/validate_seo.py`.
- If images are missing, rerun `python3 build_rotata_project.py` from the parent folder to re-scrape assets.
- If Cloudflare build fails, confirm Python 3.11+ is available and the output directory is `dist`.
