# Rotata Static Website

Premium static-first website for `rotata.es`, positioned as a B2B growth system company: **the system is the product; tools support the system**.

## Local Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
npm run build
npm run validate
```

The public build is generated into `dist/` for Cloudflare Pages.

## Structure

- `src/content/` stores editable multilingual page and blog content.
- `src/data/` stores navigation, SEO, partners, forms, cookie text and scraped source metadata.
- `src/partials/` stores reusable header, footer, forms, CTAs, cards, cookie, SEO and schema partials.
- `src/styles/` stores the premium design system.
- `src/scripts/` stores consent-aware navigation, forms, analytics, language and animation JS.
- `scripts/` stores build, SEO, schema, validation, image, content-machine and WordPress export tooling.
- `wordpress-export/rotata-theme/` is generated by `scripts/export_wordpress_theme.py`.

## Commands

```bash
npm run build              # Build static site, sitemap and schema
npm run validate           # Validate internal links and SEO basics
npm run optimize:images    # Optional Pillow-based WebP optimization
npm run og                 # Generate SVG OG images
npm run wp:export          # Generate WordPress theme folder
npm run wp:zip             # Zip WordPress theme
npm run content:ideas      # Generate article ideas/content drafts
```

## Cloudflare Pages

Build command: `npm run build`

Output folder: `dist`

Environment variables can be added in Cloudflare Pages and are read at build time.

## Integrations

Copy `.env.example` to `.env` locally or define these in Cloudflare:

- `PUBLIC_GA4_ID`
- `PUBLIC_GTM_ID`
- `PUBLIC_LINKEDIN_PARTNER_ID`
- `PUBLIC_CLARITY_ID`
- `PUBLIC_HUBSPOT_PORTAL_ID`
- `PUBLIC_HUBSPOT_FORM_ID`
- `PUBLIC_FORMS_ENDPOINT`
- `PIXABAY_API_KEY`
- `OLLAMA_HOST`

Analytics, LinkedIn Insight, Clarity and HubSpot scripts are blocked until consent allows the relevant category.

## Content Editing

Edit pages in `src/data/site.json` or `src/content/<lang>/pages.json`. Blog articles live in `src/content/es/blog/*.json` and are indexed newest first. Services, partners, cases and forms are in `src/data/`.

## SEO And Schema

SEO metadata is generated from page content plus `src/data/seo.json`. Schema modules are written to `schema/` and embedded per page at build time.

## WordPress Export

```bash
npm run wp:export
npm run wp:zip
```

The export preserves assets, CSS, JS, template parts and theme support for menus, logo, thumbnails, translations and editor styles. Future client editing should move page copy into WordPress pages/posts/custom post types rather than hard-coding it in PHP.

## Troubleshooting

- If links fail, run `python3 scripts/validate_links.py`.
- If SEO fails, run `python3 scripts/validate_seo.py`.
- If images are missing, rerun `python3 build_rotata_project.py` from the parent folder to re-scrape assets.
- If Cloudflare build fails, confirm Python 3.11+ is available and the output directory is `dist`.
